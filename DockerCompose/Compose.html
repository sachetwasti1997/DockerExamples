Basics Compose

  1. docker-compose.yml

    • Compose Yaml format has it's own versions: 1, 2, 2.1, 3, 3.1
    • YAML file can be used with docker-compose command for local docker automation
    • It can also be used with docker directly in production with Swarm
    • Some Important commands:
      • docker-compose --help
      • docker-compose.yml is the default filename, but any other file name can be used only that while running it we need to specify it after -f docker-compose -f

  2. docker compose cli

    • Not production-grade but ideal for local devlopment
    • Two most common commands are:
      • docker compose up - setup volumes/networks start all containers
      • docker compose down - stop all containers and remove cont/vol/net
    • Most important for new onboarding just with commands, if projects has Dockerfile and docker-compose.yml
      • git clone - to clone the repo
      • docker compose up
