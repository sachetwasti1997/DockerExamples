Some Swarm Commands

  • docker node ls
  • docker swarm --help
  • docker service --help
  • In a cluster we dont care about individual node, we just give requirements to the swarm in form of services.

    And them it will orchastrate, how it needs to lay all that out, on which nodes they need to be

  • To create: docker service create service(eg, alpine ping 8.8.8.8)

  • To look for all services

    docker service ls

    To look for a container in the given service

    docker service ps serviceName

  • To update something in services

    docker service update serviceId(ryomxbxesall) --update_parameter update (eg, --replica 3)
