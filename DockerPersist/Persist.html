Container LifeTime & Persisting Data

  • Containers are usually immutable and ephemeral.

    ie, unchanging and disposable

  • Containers follow "immutable infrastructure": Only redeploy containers, never change.

  • This is ideal scenario for the container, but what about databases or unique data

  • Docker gives us feature to ensure this "separation of concern".
  • This is known as persistent data.

  • There are two ways to solve the problem

    1. Volumes
    2. Bind Mounts

  • These steps prevent to loose data across container removals and allows us to attach it to whatever container we want.

  • Volumes: Make special location outside of container Union File System.

  • Bind Mounts: Sharing or mounting a host directory or file into a container
